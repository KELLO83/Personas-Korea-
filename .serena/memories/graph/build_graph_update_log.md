# 🚀 `build_graph.py` 최적화 업데이트 로그

## 📅 업데이트 일자
최근 세션 (Phase 2 진행)

## 🔧 업데이트 목적
- DB 적재 전 초기화(--reset) 지연을 통한 **다운타임 최소화**
- **누적 적재(Neo4j 데이터 유지)**를 위한 중복 UUID 필터링 로직 추가

## ✨ 주요 변경 사항

### 1. 로직 구조 리팩토링
- **기존**: main() 함수 내에서 데이터 처리 후 즉시 DB 작업 수행
- **개선**: 데이터 전처리 완료 후 별도 `_load_to_neo4j()` 함수에서 DB 작업 단일 수행

### 2. 지연 초기화 (Lazy Reset)
- `--reset` 플래그 사용 시, **데이터 적재 직전에만** Neo4j 데이터 삭제 수행
- **효과**: 전처리 시간 동안 DB 공백 상태 지속 방지 → 시스템 안정성 향상

### 3. 중복 방지 필터링
- 기존 DB에 존재하는 UUID를 사전 조회 (`_get_existing_uuids()`)
- 전처리 완료된 최종 데이터프레임에서 중복 행을 **Polars `~.is_in()`** 으로 필터링
- **효과**: 중복 데이터에 대한 불필요한 전처리/임베딩 생성 건너뜀 (성능 최적화)

### 4. 빠른 종료 (Early Exit)
- 중복 제거 후 남은 신규 데이터가 0행인 경우 **즉시 `return`**
- **효과**: 빈 DB 상태에서의 불필요한 배치 작업 방지

## 📊 파라미터 변경
- `--reset` 기본값: `False` (변경 없음, 누적 적재 모드 기본)
- `--full` 플래그: 유지 (미사용, 호환성 목적)
